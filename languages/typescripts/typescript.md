# classes Internface and Types

## Classes

- defines how **object looks like** and **what it can do**
- used for **creating objects** and services
- has access modifiers

## Interfaces

- defines contraact for the external classes to follow
- data shape for external API

## Types

- way to create more descriptive name for an **existing type or combination of types**
- primitives can be reamed as `type UserID = string;`

### Union Types 
- can take both the types 
- **narrowing** can be used to call the *number specific methods or string specific methods*.
  
  
 eg : here the weight can be number as well as string value
    `let weight : number | string;`
